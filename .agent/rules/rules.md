# ANTIGRAVITY ALWAYS-ON RULES

You must continually abide by the following rules in this repository.

## Mission
Treat this repository as a genuine, long-term product, not a one-off demo.
Continuously optimize:
- Code quality
- Runnability
- Test coverage
- Documentation completeness
- Security boundaries
- Module clarity
- Configuration template quality

## Autonomy
You are authorized to proactively:
- Inspect code structures
- Fix lint / type / test errors
- Add minimal tests
- Refactor duplicated logic
- Supplement documentation
- Supplement schemas
- Improve prompt stability
- Improve secret safety
- Improve mock completeness

## Boundaries
You must NOT:
- Expose secrets
- Print tokens / cookies / passwords in logs
- Commit real API keys
- Enable real publishing by default
- Modify files outside the repository unless approved
- Execute high-risk system operations unless approved

## Product Rules
All features must primarily serve:
- signal ingestion
- topic ranking
- draft generation
- risk guard
- approval queue
- publishing abstraction
- analytics
- learning loop

## Engineering Rules
Priority cascade:
1. Main pipeline is runnable
2. Tests pass
3. Configuration is clear
4. Module boundaries are stable
5. Documentation is complete
6. Extensible capabilities

## Secret Safety
Anywhere secrets are handled:
- Only reference env names
- Create example files
- Use redaction helpers
- Forbid injecting secret values into docs / code / logs / tests

## Change Strategy
When modifying code:
- Submit small steps
- Single purpose updates
- Verifiable
- Rollback capable

## Validation
After making changes, default to executing:
- install / dependency check
- typecheck
- tests
- minimal run path
- docs consistency check

## Escalation
The following operations MUST request approval first:
- Using real social media APIs
- Modifying system-level configs
- Writing files outside the repo
- Globally installing dependencies
- Accessing private directories
- Executing destructive shell commands

## Communication Style
Outputs should be:
- Direct
- Concise
- Action-oriented
- Clear about incomplete tasks
- Do not exaggerate completion
